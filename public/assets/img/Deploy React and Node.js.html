To deploy your React.js frontend and Node.js (Express) backend using Hostinger and AWS, follow these steps:

1. Prepare your React and Node.js Applications

Before deployment, ensure that both your React frontend and Node.js backend are ready for production:

  • For React.js (Frontend):
    • In your React project folder, run:
      bash
      npm run build
      This command generates the production-ready files in a build/ folder.
  • For Node.js (Backend):
    • Ensure your server code is prepared and running properly on your local machine.

2. Deploy Frontend on Hostinger

Hostinger supports static file hosting, perfect for deploying the React build.

Steps to deploy React frontend:

  1. Log in to your Hostinger account.
  2. Go to the File Manager under the Hosting section or use an FTP client like FileZilla.
  3. Upload React Build Files:
    • Navigate to the public_html folder (or the root directory where your domain points).
    • Upload the contents of the build/ folder from your React app.
  4. Configure domain (if necessary):
    • Make sure your domain is correctly pointing to this folder. If not, you can manage domains under Hostingers control panel.

After deployment, your React frontend should be accessible from your domain.

3. Deploy Backend (Node.js) on AWS

Since Hostinger doesnt support Node.js hosting directly, you can deploy your Node.js (Express) backend on AWS.

Steps to deploy Node.js backend on AWS EC2:

  1. Launch an EC2 Instance:

    • Sign in to the AWS Management Console.
    • Go to the EC2 service and click on Launch Instance.
    • Choose an appropriate instance type (e.g., t2.micro if its a lightweight application).
    • Choose Ubuntu as the AMI (Amazon Machine Image).
    • Configure security groups to allow HTTP, HTTPS, and any necessary ports (like port 3000 for your Node.js app).

  2. Connect to the Instance:

    • After launching the instance, connect via SSH using:
      bash
      ssh -i /path/to/your-key.pem ubuntu@your-ec2-public-ip

  3. Set up the environment:

    • Install Node.js:
      bash
      sudo apt update
sudo apt install nodejs npm
    • Install PM2 to manage the Node.js process:
      bash
      sudo npm install -g pm2
    • Install Nginx for reverse proxy:
      bash
      sudo apt install nginx

  4. Upload your Node.js project:

    • Clone your project from GitHub or upload your code using SCP/FTP.

  5. Run your Node.js App:

    • Navigate to your Node.js project folder and install dependencies:
      bash
      npm install
    • Start the app using PM2:
      bash
      pm2 start app.js
pm2 startup
pm2 save

  6. Configure Nginx:

    • Open the Nginx config file:
      bash
      sudo nano /etc/nginx/sites-available/default
    • Add a reverse proxy to your Node.js server in the config:
      nginx
      server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
    • Restart Nginx:
      bash
      sudo systemctl restart nginx

4. Configure DNS

  • Point your domain (e.g., api.yourdomain.com for the backend) to the AWS EC2 instance's public IP in your DNS settings.
  • For the frontend, ensure the domain is correctly configured to the Hostinger server.

5. Secure the Application

  • SSL/TLS Certificates: Install SSL certificates on both Hostinger and your AWS EC2 instance to secure your website.
    • For Hostinger, you can use their SSL setup tool.
    • For AWS, use Certbot to install a Lets Encrypt SSL certificate.

Once everything is configured, your React frontend should be accessible from Hostinger, and your Node.js backend from AWS, both running under your domain.

To connect to a MongoDB instance running on another computer, you'll need to configure both the MongoDB server and your local computer to allow remote access.

Steps to Allow Remote Access to MongoDB

1. Configure MongoDB on the Server PC (Where MongoDB is Installed):

  1. Edit the MongoDB Configuration File:

    • On the server PC (where MongoDB is running), open the MongoDB configuration file. This file is usually located at /etc/mongod.conf (on Linux) or C:\Program Files\MongoDB\Server\<version>\bin\mongod.cfg (on Windows).
    • In the configuration file, find the net section and modify it to allow access from other IP addresses:
      yaml
      net:
  bindIp: 127.0.0.1,0.0.0.0
  port: 27017
      • 127.0.0.1 allows local connections.
      • 0.0.0.0 allows remote connections from any IP. If you want to restrict access to specific IP addresses, you can replace 0.0.0.0 with the IP address of your other PC.

  2. Restart MongoDB:

    • After modifying the configuration file, restart MongoDB for the changes to take effect:
      • On Linux:
        bash
        sudo systemctl restart mongod
      • On Windows: Restart the MongoDB service using the Services app or Command Prompt:
        bash
        net stop MongoDB
net start MongoDB

2. Allow MongoDB Through the Firewall on the Server PC:

  • If your server PC has a firewall enabled, ensure that MongoDBs default port (27017) is allowed for remote connections.

  • On Linux (Ubuntu):

    bash
    sudo ufw allow 27017

  • On Windows:

    • Open Windows Defender Firewall.
    • Go to Advanced Settings.
    • In Inbound Rules, create a new rule allowing port 27017 for TCP connections.

3. Ensure MongoDB Authentication (Optional but Recommended):

  • To enhance security, set up MongoDB authentication (username and password) if not already done, so that only authenticated users can connect to the database.

  • You can create a MongoDB user with access privileges by connecting to MongoDB locally on the server PC and running the following:

    bash
    use admin
db.createUser({
  user: "yourUsername",
  pwd: "yourPassword",
  roles: [{ role: "readWrite", db: "yourDatabase" }]
})

4. Connect to MongoDB from the Other PC (Client PC):

  • On the client PC (where you want to connect to MongoDB), youll need the IP address of the server PC.

  • Install MongoDB tools on the client PC (if not already installed) by downloading the MongoDB shell (mongosh) from the MongoDB website.

  • Then, connect to the remote MongoDB instance using the following command:

    bash
    mongosh "mongodb://yourUsername:yourPassword@server-PC-IP:27017/yourDatabase"

  • Replace:

    • yourUsername and yourPassword with the MongoDB credentials.
    • server-PC-IP with the IP address of the server PC.
    • yourDatabase with the database name you want to access.

5. Test the Connection:

  • If the connection is successful, youll be able to interact with the MongoDB database remotely.
  • You can also use GUI tools like MongoDB Compass or database clients like Robo 3T to connect using the same connection string.

Additional Tips:

  • Security: For production, it's recommended to enable authentication and use a VPN or SSH tunneling if accessing MongoDB over the internet. Also, consider limiting bindIp to specific IP addresses rather than using 0.0.0.0.

  • SSH Tunnel (Optional but Secure): If you want to securely connect to MongoDB without opening up port 27017 to the world, you can use SSH tunneling:

    • On the client PC, create an SSH tunnel to the server PC:
      bash
      ssh -L 27018:localhost:27017 your-username@server-PC-IP
    • Then, connect to MongoDB on localhost:27018 on the client PC:
      bash
      mongosh "mongodb://yourUsername:yourPassword@localhost:27018/yourDatabase"

This approach secures the connection while avoiding exposing MongoDB directly over the network.

bash

      bash

javascript

const { Pool } = require('pg');

// PostgreSQL connection settings
const pool = new Pool({
  user: 'your_user',

Steps to Create and Connect a MongoDB Database in Node.js (Express)

Step 1: Set Up Your Node.js Express Project

If you haven't already created an Express project, you can initialize it using the following commands:

bash
mkdir my-express-app
cd my-express-app
npm init -y
npm install express mongoose

This will install the express framework and mongoose, which is a popular ODM (Object Data Modeling) library for MongoDB.

Step 2: Install MongoDB and Mongoose

You'll need MongoDB running locally or on a remote server. If MongoDB is already set up, you can proceed with adding mongoose to your project, which will simplify database interactions.

Install mongoose:

bash
npm install mongoose

Step 3: Set Up MongoDB and Mongoose in Your Express App

Create a db.js file to manage your MongoDB connection. Use Mongoose to connect to your MongoDB instance.

javascript
// db.js

const mongoose = require('mongoose');

// MongoDB connection string
const mongoURI = 'mongodb://localhost:27017/your_new_database'; // Replace with your new database name

// Connect to MongoDB
const connectDB = async () => {
  try {
    await mongoose.connect(mongoURI, {
      useNewUrlParser: true,
      useUnifiedTopology: true,
    });
    console.log('MongoDB connected successfully');
  } catch (err) {
    console.error('Failed to connect to MongoDB', err);
    process.exit(1); // Exit process with failure
  }
};

module.exports = connectDB;
  • Replace your_new_database with the name of the new MongoDB database you want to create.
  • When you first connect using mongoose.connect, MongoDB will automatically create the new database if it doesn't already exist.

Step 4: Initialize the MongoDB Connection in Your Express App

In your main Express application file (app.js or server.js), import the connectDB function and call it to establish a connection to MongoDB.

javascript
// server.js or app.js

const express = require('express');
const connectDB = require('./db');

const app = express();
const port = 5000;

// Connect to MongoDB
connectDB();

// Middleware
app.use(express.json());

// Sample route
app.get('/', (req, res) => {
  res.send('Hello, MongoDB and Express!');
});

// Start the server
app.listen(port, () => {
  console.log(`Server is running on port ${port}`);
});

Step 5: Create a Mongoose Model

To add data to your new MongoDB database, you need to define a Mongoose model. Models in Mongoose represent collections in MongoDB.

Create a new file called models/User.js to define a User model:

javascript
// models/User.js

const mongoose = require('mongoose');

// Define the User schema
const userSchema = new mongoose.Schema({
  name: {
    type: String,
    required: true,
  },
  email: {
    type: String,
    required: true,
    unique: true,
  },
  password: {
    type: String,
    required: true,
  },
  date: {
    type: Date,
    default: Date.now,
  },
});

// Create the User model
const User = mongoose.model('User', userSchema);

module.exports = User;

Step 6: Add Data to the New MongoDB Database

Now you can create a new route in Express to add data (documents) to your new MongoDB database. In your server.js or app.js, add the following route:

javascript
const User = require('./models/User');

app.post('/adduser', async (req, res) => {
  const { name, email, password } = req.body;

  try {
    const newUser = new User({

  }

Run your server:

node server.js

json

If successful, youll see the user added to your MongoDB database.

You can verify the new MongoDB database and collection by connecting to MongoDB using the Mongo shell or MongoDB Compass:

bash
mongo
use your_new_database
show collections
db.users.find().pretty()

This will show the users collection and any documents youve added.

Summary:

  1. Install mongoose and connect to MongoDB using the connection string for your new database.
  2. Define your models using Mongoose schemas.
  3. Create API routes in Express to add data to MongoDB.
  4. Test the API to ensure that new data is added to the new database.

This setup allows you to create and interact with a MongoDB database using Node.js and Express.
